# Game Member Generator

スポーツ（テニスやバドミントンなど）の練習会で、参加メンバを元に自動で試合の組み合わせ（マッチメイキング）を行う Flutter アプリケーションです。

## 主な機能

- **メンバ管理**: 
  - 登録、編集、削除、および「よみがな」による正確な五十音順ソート。
  - **リアルタイム検索**: 名前やよみがなでメンバを素早く絞り込み。
  - **参加状況の切り替え**: チェックボックスで本日の参加メンバをワンタップで切り替え。
  - **絶対休みフラグ**: 体調不良や休憩が必要なメンバを「必ず休み」としてマークし、選出から除外可能。
  - **一括インポート・エクスポート**: クリップボード経由でメンバリストを JSON 形式で簡単にバックアップ・移行。
- **自動マッチメイキング (Balanced Matching)**:
  過去の試合履歴を多角的に分析し、最適な組み合わせをバックグラウンドで高速に算出（Isolate使用）します。
  - **評価基準（優先度順）**:
    1. **休みが連続しない**: 前回お休みだったプレイヤーを最優先で選出。
    2. **種目のバランス**: 男子/女子ダブルスと混合ダブルスの出場割合を均等化。
    3. **ペアの新鮮さ**: 過去に組んだ回数が少ない組み合わせを優先。
    4. **対戦相手の新鮮さ**: 過去に対戦した回数が少ない組み合わせを優先。
  - **バリデーション**: 人数が不足している場合は事前に警告を表示し、生成を防止。
- **試合履歴の管理**:
  - 生成された組み合わせをセッションごとに保存。
  - **再生成機能**: 履歴にある試合を、最新のメンバ状況や異なるマッチ構成でやり直し可能。
  - 統計情報（出場・お休み回数、種目別回数、ペア結成回数）を詳細に表示。
- **洗練された UI**:
  - カード形式による見やすい試合表示。
  - レスポンシブ設計：タブレットや横画面では対戦カードを2列で表示。
  - 柔軟な手動調整：長押し＆タップで試合中のメンバを自由に入れ替え可能。
- **SQLite 永続化**: すべてのデータは端末内に安全に保存されます（DB Version 3）。

## 技術スタック

- **Flutter / Dart**: クロスプラットフォーム UI フレームワーク。
- **sqflite (FFI)**: ローカルデータベースによる永続化。Linux デスクトップ環境にも対応。
- **Clean Architecture**: 疎結合でメンテナンス性の高いコード構造。
- **Isolate (compute)**: 重い組み合わせ計算を別スレッドで実行し、UI のフリーズを防止。

## 開発環境のセットアップ (Linux)

Linux 環境で実行する場合、SQLite (FFI) を動作させるために以下のパッケージが必要になる場合があります。

```bash
sudo apt-get update
sudo apt-get install libsqlite3-dev
```
FFI の初期化は `lib/main.dart` で自動的に行われます。
